# 三人交接规范

## 成员 A：后端/部署

负责 `backend/`、`deploy/`、`docs/API.md`、`scripts/init_db.sql`。

交接给 B：接口地址、请求响应字段、错误码、Mock 数据。

交接给 C：RAG 调用格式、知识库上传/检索接口、日志字段。

## 成员 B：前端/数字人/大屏

负责 `frontend/`，重点是游客端、管理端、数字人状态机、口型同步、地图和数据大屏。

交接给 A：页面字段需求、接口联调问题。

交接给 C：表情枚举、话术长度、路线卡片格式。

## 成员 C：AI/知识库/测试文档

负责 `ai_service/`、`data/`、测试集、Prompt、报告和 PPT 内容。

交接给 A：AI 服务输入输出、路线推荐函数、情绪分析格式。

交接给 B：演示问题、景点讲解词、推荐路线文案。
